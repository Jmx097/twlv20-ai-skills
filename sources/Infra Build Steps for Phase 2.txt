### Step-by-step

1. **First-login hygiene**: `ssh root@<ip>`, `apt update && apt -y full-upgrade`, install ufw/fail2ban/curl, set hostname `twlv20-prod`, timezone UTC.

2. **Create non-root sudo users**: `adduser keeper/jon`, add to sudo group, setup `.ssh/authorized_keys`, test logins.

5. **Caddy install**: Official apt repo, confirm service running.

6. **Caddyfile baseline**: Routes `paperclip./claw./infisical.` to localhost ports, JSON logs, auto TLS.

7. **fail2ban — SSH jail**: SSHD jail with maxretry=3.

8. **fail2ban — Caddy jail**: Custom filter for 401/403/404 status codes in Caddy JSON logs.

9. **Final lockdown**: Lock root password, enable services.

### Acceptance Criteria

## Task 2 — Postgres 16 + pgvector install

**Priority:** Urgent | **Status:** TO DO | **Owner:** Keeper | **Depends on:** Task 1 | **Blocks:** Tasks 3, 4

### Outcome

PostgreSQL 16 with pgvector running on the hardened droplet, listening on the Unix socket and `127.0.0.1` only, with a non-superuser application role and tuning matched to 16 GB RAM. This is the data spine the Run-Log, tenant schema, and Infisical all sit on.

### Pre-flight

- Task 1 (Droplet hardening) shipped.

- DNS not relevant — Postgres never speaks publicly.

### Step-by-step

1. **Add the PGDG official apt repo (deb822 format)**

   ```bash

   sudo apt -y install curl ca-certificates

   sudo install -d /usr/share/postgresql-common/pgdg

   sudo curl -o /usr/share/postgresql-common/pgdg/apt.postgresql.org.asc \

     --fail https://www.postgresql.org/media/keys/ACCC4CF8.asc

   sudo tee /etc/apt/sources.list.d/pgdg.sources << 'EOF'

   Types: deb

   URIs: https://apt.postgresql.org/pub/repos/apt

   Suites: noble-pgdg

   Architectures: amd64

   Components: main

   Signed-By: /usr/share/postgresql-common/pgdg/apt.postgresql.org.asc

   EOF

   sudo apt update

   ```

2. **Install Postgres 16 + pgvector**

   ```bash

   sudo apt -y install postgresql-16 postgresql-client-16 postgresql-16-pgvector

   psql -V  # confirm 16.x

   ls -l /usr/lib/postgresql/16/lib/vector.so  # confirm pgvector library present

   ```

3. **Create the application role and the main database**

   ```sql

   sudo -u postgres psql << 'SQL'

   CREATE ROLE twlv20_app LOGIN PASSWORD :'app_pw';

   ALTER ROLE twlv20_app NOSUPERUSER NOCREATEDB NOCREATEROLE NOBYPASSRLS;

   CREATE DATABASE twlv20 OWNER twlv20_app;

   \c twlv20

   CREATE EXTENSION IF NOT EXISTS vector;

   CREATE EXTENSION IF NOT EXISTS pgcrypto;

   SQL

   ```

4. **Lock listener and `pg_hba.conf` to local-only** (configs for `postgresql.conf` and `pg_hba.conf`)

5. **Tune for 16 GB RAM** (tuning parameters in `conf.d/twlv20-tuning.conf`)

6. **Smoke-test pgvector** (create table, insert vectors, query L2 distance)

7. **Service health check**

   ```bash

   sudo systemctl status postgresql@16-main

   sudo -u postgres psql -c "SELECT version();"

   ```

### Acceptance Criteria

8 points: version 16.x, pgvector installed, local-only bind, role config, tuning values, smoke test passes, password stored.

## Task 3 — Tenant schema + RLS migration (per Plinko spec)

**Priority:** Urgent | **Status:** TO DO | **Owner:** Keeper | **Depends on:** Task 2 + schema.sql | **Blocks:** Tenant stubs, DB writes

### Outcome

Plinko's `infra/schema.sql` migration is applied to the `twlv20` database. Every tenant-scoped table carries `tenant_id`, has RLS enabled, has policies that key reads/writes on `current_setting('app.tenant_id')`, and `FORCE ROW LEVEL SECURITY` is set so the table owner is also subject to policies. Four tenant rows exist. An RLS isolation test passes for all four tenants.

### How RLS Will Be Used (so Keeper isn't guessing)

The agent runtime (`twlv20_app` role, `NOBYPASSRLS`) opens a Postgres connection per request and *immediately* runs `SELECT set_config('app.tenant_id', '<tenant-uuid>', false);` before any query. Every policy on every tenant-scoped table compares `tenant_id::text` to that GUC. If the runtime forgets to set the GUC, queries return zero rows (the GUC defaults to empty — fail-closed, not fail-open). Plinko owns the runtime side; Keeper's job is to make sure the SQL side is bulletproof.

### Pre-flight

- Postgres + pgvector done.

- `infra/schema.sql` from Plinko required.

### Tables

`tenants`, `users`, `credentials`, `skills`, `kb_docs`, `agents`, `runs`, `approvals`, `artifacts` (`tenants` table excepted from RLS).

### Steps

1. Apply: `psql ... -f infra/schema.sql`

2. Verify schema (`\dt`, `\d runs`)

3. Check RLS (`SELECT ... FROM pg_class`)

4. Verify policies (`\d+ <table>`)

5. Insert 4 tenants: `'pure-peptide'`, `'agere-sciences'`, `'sru'`, `'twlv20-internal'`

6. Run isolation test `infra/tests/rls_isolation.sql`

7. Test all 4×4 pairs

### Acceptance Criteria

All tables present, RLS/force_rls true, policies correct, 4 UUIDs captured, tests pass.

## Task 5 — Weekly encrypted Postgres snapshots → S3 / Backblaze B2

**Priority:** High | **Status:** TO DO | **Owner:** Keeper | **Depends on:** Tasks 2, 4 | **Review Gate:** Jon performs restore drill

### Outcome

Every Sunday at 03:00 UTC, the droplet runs `pg_dump` over both `twlv20` and `infisical` databases in custom format, encrypts each dump with GPG (symmetric, AES256), uploads to Backblaze B2 via rclone, retains the 8 most recent weekly snapshots, and emails on failure. A documented restore procedure has been exercised end-to-end on a scratch host.

### Why This Shape

- `pg_dump -Fc` (custom format) — compressed by default, supports parallel restore via `pg_restore -j`, and lets you cherry-pick tables on restore.

- **GPG symmetric (AES256), not asymmetric** — one passphrase to manage, simpler operations. The passphrase lives in Infisical (Phase 1, post-Task 4) under `backup.gpg.passphrase` AND offline.

- **Backblaze B2**: cheaper than S3, rclone native support.

- Weekly cadence matches contract §4.

### Pre-flight

- Tasks 1, 2, 4 complete (Postgres + Infisical).

- Confirm B2 bucket access with Jessie.

### Step-by-step (9 steps)

1. Install rclone/gpg/postgresql-client-16

2. Configure rclone for B2 (`b2-twlv20`)

3. Generate/store GPG passphrase (Infisical + offline + paper)

4. Backup script `/usr/local/sbin/twlv20-backup.sh` (dumps → encrypts → uploads → retention)

5. Postgres peer auth verification

6. systemd timer (weekly Sun 03:00 UTC)

7. Smoke test

8. **Restore drill** (acceptance): pull/decrypt/restore/verify row counts

9. Failure alert verification

### Script Overview

Bash script fetches passphrase from Infisical, dumps both DBs, GPG encrypts, rclone uploads, prunes to 8 newest per DB, alerts on failure.

### Acceptance Criteria

7 points: first run success, timer scheduled, retention logic, restore validation, alerts work, passphrase secured, physical separation.
